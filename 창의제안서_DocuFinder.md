## 현황 및 문제점

 **"그 문서 어디 있었지?" — 매일 반복되는 문서 검색의 비효율**

 - 감사 대응, 인수인계, 유사 공문 작성 등 과거 문서를 찾아야 할 때마다 폴더를 하나씩 열어 확인하는 수작업 반복
 - 정작 본 업무에 집중할 시간이 줄고, "분명 있었는데" 찾지 못해 처음부터 재작성하는 경우도 빈번

 **Windows 기본 검색은 HWPX(한컴오피스) 문서 내용을 검색할 수 없음**

 - 공문서의 대다수가 HWPX 형식이나, Windows 기본 검색은 파일명만 검색 가능
 - HWPX 문서 내용을 검색할 수 있는 도구가 사실상 존재하지 않음

 **민간에서는 AI 검색이 일상이 되었으나, 망분리 환경에서는 활용 불가**

 - 행정망에서 클라우드 AI 서비스(ChatGPT, Copilot 등) 접속 차단
 - 내부 문서의 개인정보·정책자료 특성상 외부 서버 전송은 보안상 불가

 ※ 공무원이 AI 혁신의 혜택에서 사실상 소외되고 있는 상황

## 개선방안 (실행내용)

 **PC에서 직접 실행되는 AI 임베딩 엔진을 탑재한 문서검색 시스템(DocuFinder) 자체 개발**

 - HWPX, DOCX, XLSX, PDF, TXT 등 공문서 형식을 모두 파싱하여 내용 기반 검색 실현
 - 오프라인 AI 모델을 앱에 내장, CPU만으로 의미 기반 검색 수행
 - "예산 절감"으로 검색하면 "비용 효율화"도 함께 발견 (키워드 불일치 문제 해결)

 **하이브리드 검색 엔진**

 - 키워드 정확 검색(SQLite FTS5) + AI 의미 검색(벡터 HNSW)을 RRF 알고리즘으로 결합
 - 한국어 형태소 분석 엔진 탑재로 한국어 검색 정확도 대폭 향상

 **100% 오프라인 보안 설계**

 - 네트워크 모듈 자체를 제거하여 외부 통신 원천 차단
 - 문서 내용이 PC 밖으로 유출될 가능성 없음
 - 압축 폭탄 방어, SQL 인젝션 방지, 파싱 타임아웃 등 보안 위협 대응 완료

 **실무 편의기능 구현 완료**

 - 폴더 내 문서 변경 자동 감지 → 실시간 증분 인덱싱
 - 파일명 즉시 검색, 검색 결과 CSV 내보내기, 즐겨찾기 폴더
 - MSI 설치파일(약 50MB)로 IT 자산관리 시스템을 통한 일괄 배포 가능

 ※ 2026년 2월 현재 개발 완료·실행 중, 약 1,700개 파일 테스트에서 92% 파싱 성공

## 기대효과

 **업무 효율화**: 폴더 탐색·파일 열기를 반복하던 문서 검색이 키워드 한 번으로 해결

 - 감사 대응, 인수인계, 유사 공문 참고 등 반복 업무에서 시간 절감
 - 키워드를 정확히 기억하지 못해도 의미 기반으로 문서 발견 가능

 **비용 없는 혁신**: 자체 개발·오픈소스 기반으로 별도 예산 투입 없이 도입 가능

 - 외부 용역 개발이나 상용 SW 구매 없이 즉시 배포 가능한 완성된 시스템
 - 서울시 전 자치구 확산 시에도 추가 라이선스 비용 없음

 **망분리 환경 AI 혁신 선도**: 클라우드 없이도 AI 검색이 가능함을 실증

 - 향후 AI 문서 요약, 자동 분류 등으로 확장 가능한 플랫폼 기반
 - 서울시 전 자치구(약 3만 명) 확산 시 공무원 문서 검색 업무 전면 혁신

 **정보 보안 강화**: 외부 클라우드 미사용으로 문서 유출 위험 원천 차단

 - 개인정보 포함 문서도 안전하게 검색 가능, 사이버안보 정책과 일치
