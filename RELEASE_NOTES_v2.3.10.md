# Anything v2.3.10

**내 PC 문서를 통째로 검색하는 로컬 검색 엔진**
파일명 몰라도, 열어보지 않아도. 문서 **내용**으로 찾습니다.

> v2.3.9 설치 후 재실행이 안 되던 치명적 회귀를 잡은 핫픽스입니다. v2.3.9 사용자는 이 버전으로 바로 업데이트하세요.

---

## 설치하기

1. 아래 **Assets** 에서 `Anything_2.3.10_x64_en-US.msi` 다운로드
2. MSI 파일 더블클릭 → 설치 (UAC 한 번)
3. 시작 메뉴에서 **Anything** 실행

### 시스템 요구사항
- Windows 10 (21H2 이상) 또는 Windows 11
- RAM 8GB 이상 (16GB 권장)
- 디스크 여유 1GB 이상
- 최초 실행 시 인터넷 연결 필요 — ONNX 모델(~420MB) 자동 다운로드

WebView2, VC++ 런타임은 설치파일에 포함되어 있어 별도 설치 불필요.

### 보안 경고가 뜰 때

코드서명 인증서가 없어서 경고가 보일 수 있습니다. 악성코드가 아닙니다 — 소스는 전부 이 저장소에서 확인 가능.

- **Windows의 PC 보호 파란 창**: "추가 정보" 클릭 → "실행"
- **스마트 앱 컨트롤 차단 (Win11)**: MSI 파일 우클릭 → 속성 → "차단 해제"
- **Edge/Chrome 다운로드 차단**: 다운로드 창에서 "..." → "유지"
- **백신 자동 격리**: 보호 기록에서 **복원** 또는 실시간 감시 잠시 끄고 재시도

## 빠른 시작

### 1. 폴더 등록
앱 실행 후 좌측 **"폴더 추가"** 로 검색할 문서 폴더를 선택하세요. 자동으로 인덱싱이 시작됩니다.

- 전체 드라이브(`C:\`)도 등록 가능 — 시스템 폴더/빌드 산출물은 자동 제외
- `.gitignore` 있는 프로젝트 폴더는 그 규칙을 자동 존중

### 2. 검색
검색창에 키워드를 치면 **문서 내용**에서 결과를 찾아줍니다.

| 모드 | 쓰임 |
|------|------|
| 키워드 | 기본값. 정확한 단어 매칭 (1초 이내) |
| 파일명 | Everything 스타일. 인덱싱 끝나기 전에도 즉시 사용 |
| 하이브리드 | 키워드 + 의미 검색 결합 (시맨틱 ON 필요) |
| 시맨틱 | 의미 기반 유사 문서 (시맨틱 ON 필요) |

### 3. AI 기능 (선택)
설정에서 Gemini API 키를 입력하면 사용할 수 있습니다. 끄면 네트워크 통신 제로 — 완전 오프라인 검색 도구로 쓸 수 있습니다.

- **AI 질문답변**: "2026년 예산 얼마야?" 같은 자연어 질문 → 근거 문서/페이지와 함께 답변
- **AI 요약**: 파일 우클릭 → 요약. 계약서·보고서·회의록 핵심 뽑기

### 4. 문서 버전 관리
같은 문서의 여러 버전(`최종.docx`, `최최종.docx`, `v2.docx` 등)은 자동으로 묶여 대표 1개만 표시됩니다. 뱃지 클릭으로 다른 버전 탐색, 버전 간 내용 비교도 가능.

## v2.3.9 에서 업그레이드 시 주의

이번 릴리즈는 v2.3.9 의 마이그레이션 결함을 잡은 긴급 패치입니다. **별도 재설치 없이** 그냥 이 MSI 로 덮어 설치하면 됩니다.

이미 앱이 안 켜지는 증상을 겪고 있다면:
1. 앱 종료 (작업 관리자에서 `anything.exe` 완전히 끄기)
2. 이 버전 MSI 설치
3. 실행 → 자동 복구. 기존 인덱싱은 유지되며, 벡터 인덱스만 다음 백그라운드 재색인 주기에 자동 교체됩니다.

### Fixed
- **[Critical] v2.3.9 마이그레이션 v14 치명 결함** — v14 가 지우려던 `vectors.usearch.map` 파일은 실제 존재하지 않는 이름이었습니다. `Path::with_extension("map")` 은 기존 `.usearch` 확장자를 **교체**하므로 실제 맵 파일명은 `vectors.map`. 결과적으로 마이그레이션이 본체(`.usearch`)만 지우고 맵(`.map`)은 그대로 두어, 다음 부팅의 `VectorIndex::new` 가 짝이 맞지 않는 mmap 위에서 usearch FFI segfault 가능성. 파일명 목록을 실제 구성에 맞추고 save 중간산출물(`.tmp`)까지 보수적으로 함께 회수.
- **[Build] Rust 1.95 신규 clippy 린트 4건** — `type_complexity` 2건은 type alias 로 분리, `needless_range_loop` 는 `enumerate()` 로 전환, `needless_borrows_for_generic_args` 는 불필요한 `&` 제거. CI 빌드 차단 해제.

### Migration
- 스키마 v14 재진입 — 이미 v2.3.9 에서 v14 가 "성공"으로 기록됐다면 마이그레이션이 재실행되지 않습니다. 남아 있을 수 있는 `vectors.map` 은 `VectorIndex::new` 의 이중 폴백(mmap view → full load → 빈 인덱스 회수)으로 부팅 안정성이 확보되며, 다음 벡터 재색인 주기에 정상 파일로 교체됩니다.
